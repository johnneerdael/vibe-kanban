## Prerequisites

Before installing Vibe Kanban, ensure you have:

- **Docker**: [Install Docker](https://docs.docker.com/get-docker/)
- **Docker Compose**: [Install Docker Compose](https://docs.docker.com/compose/install/)
- **Coding agent authentication**: Authenticate with your preferred coding agents outside of Vibe Kanban (or inside the container if they are CLI-based).

## Safety Notice

<Warning>
Vibe Kanban runs AI agents with --dangerously-skip-permissions/--yolo flags by default so they can work autonomously without constant approval prompts. Each task runs in an isolated git worktree, preventing agents from interfering with each other.
Agents can still perform system-level actions, so review their work and keep backups.
</Warning>

## Installation & Setup

The recommended way to run Vibe Kanban is using the official Docker image.

<Steps>
<Step title="Create a docker-compose.yml">
  Create a new directory for Vibe Kanban and add a `docker-compose.yml` file:

  ```yaml
  services:
    vibe-kanban:
      image: johnneerdael/vibe-kanban:main
      ports:
        - "3000:3000"
      volumes:
        - ./data:/data
        - ./repos:/repos
      environment:
        - RUST_LOG=info
        - AUTH_USER=admin    # Optional: Enable Basic Auth
        - AUTH_PASS=password # Optional: Enable Basic Auth
      restart: unless-stopped
  ```
</Step>

<Step title="Launch the application">
  In your terminal, run:

  ```bash
  docker compose up -d
  ```

  <Check>
  The application will start on port 3000. Access it at [http://localhost:3000](http://localhost:3000).
  </Check>
</Step>

<Step title="Complete initial setup">
  Complete the setup dialogs in the web UI to configure your coding agent and editor preferences.
</Step>

<Step title="GitHub Integration">
  Vibe Kanban uses the [GitHub CLI](https://cli.github.com/) for creating pull requests. To authenticate inside the container:

  ```bash
  docker compose exec vibe-kanban gh auth login
  ```
</Step>
</Steps>

## Persistence

Vibe Kanban persists data in two main directories on your host machine:

- `./data`: Stores the SQLite database, configuration files, and GitHub CLI authentication.
- `./repos`: Stores the git worktrees and repositories managed by the application.

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `PORT` | Internal server port | `3000` |
| `HOST` | Internal bind address | `0.0.0.0` |
| `AUTH_USER` | Basic Auth username | (None) |
| `AUTH_PASS` | Basic Auth password | (None) |
| `RUST_LOG` | Logging level (debug, info, warn, error) | `info` |
